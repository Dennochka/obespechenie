## Ответы на вопросы
1. Параметр - техническая характеристика окружения, на котором проводится тестирование ПО. Они содержат значения и помогают избежать дублирования тест-кейсов. Для создания параметров: Войдиет в систему, с ролью **Администратор**, затем в верхней навигационной панели выберите **Настройки Проектов** -> В навигационном меню слева выберите  **Параметры**. Отобразится список параметров -> Нажмите **Создать параметр** -> Введите данные параметра (Название, Проекты, в которых параметр будет использоваться, Значения параметра) -> Нажмите **Сохранить** -> Чтобы отредактировать параметр, нажмите значок **Действия** справа от его названия, выберите **Редактировать**, затем внесите необходимые изменения и сохраните их.

2. Поле продолжительность отвечает за время прохождения тест-кейса.

3. Ссылки в проекте указываются для того, чтобы ссылаться на задачи, дефекты и связанные рабочие элементы, а также отслеживать статус связанных задач и дефектов. Для добавления ссылки: Откройте проект -> Откройте рабочий элемент, в который хотите добавить ссылку -> В навигационном меню слева перейдите в раздел **Ссылки** -> Введите тип и URL ссылки, которую хотите добавить в рабочий элемент -> Нажмите **Добавить**. Ссылка отобразится в таблице.

5. Общий шаг - Повторяющиеся действия в нескольких тест-кейсах. Для добавления общего шага: Отпройте проект, перейдите в раздел **Библиотека тестов** -> Создайте новый тест-кейс или откройте существующий -> В пред-/постусловиях или шагах теста нажмите **Добавить** -> Нажмите **Добавить общий шаг** в выпадающем меню в правой части строки шага -> Выберите нужный общий шаг из списка или найдите его с помощью строки поиска -> Нажмите **Добавить**.

6. Конфигурационный файл для импорта тест-кейсов. Импортер - утилита для переноса файлов с тестово документацией из любых сторонних TMS-систем в TestIT. Чтобы загрузить тестовую документацию в TestIT с помощью импортера: Создайте конфигурационный файл ``` tms.config.json ``` и сохраните его в папке с импортером. Настройте соединение в файле ``` tms.config.json ```. -> Запустите ```.exe```-файл экспортера и дождитесь окончания импорта. По окончании импорта тестовая документация будет пересена в TestIT.

7. Атрибуты - задают нужные вам значения для тестов и тест-планов, они также помогают сделать поиск с помощью фильтра более точным. Атрибуты используются в проктах,  для тест-кейсов или тест-планов. Чтобы создать атрибут: Используя аккаунт с правами администратора, нажмите **Настройки проектов** на верхней навигационной панели -> В навигационном меню слева выберите **Атрибуты**, убедитесь, что вы находитесь на вкладке **Глобальные**, и нажмите **Добавить** -> В открывшимся окне введите название нового атрибута, выберите его тип: **Строка, Дата, Варианты на выбор, Множественный выбор, Пользователь и Чекбокс** -> Вы можете сделать атрибут обязательным для всех проектов. поставив флажок **Обязательный атрибут** -> Нажмите **Сохранить**.
